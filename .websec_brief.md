## 1. Project Name & Type

**OWASP Juice Shop** - Web Application (Security Training Platform / Intentionally Vulnerable Application)

## 2. Main Purpose

This is a deliberately insecure web application designed for security education, awareness training, and penetration testing practice. It serves as an OWASP project for learning about modern web application security vulnerabilities through hands-on experience. The application functions as an e-commerce storefront (selling juice products and merchandise) while containing numerous security flaws that users must identify and exploit to complete challenges.

## 3. Technology Stack

**Backend:**

- Node.js (v20-24) with TypeScript 5.3.3
- Express.js 4.21.0 web framework
- Sequelize 6.37.3 ORM with SQLite 5.1.7 database
- JWT (jsonwebtoken) for authentication
- Socket.io 3.1.2 for real-time communication
- Helmet, CORS, body-parser, cookie-parser middleware
- Multer for file uploads
- Web3 4.13.0 & ethers 6.13.2 for blockchain integration
- Prometheus 14.2.0 for metrics collection
- Swagger UI for API documentation

**Frontend:**

- Angular 20.1.0 with TypeScript 5.8.2
- Angular Material 20.1.0 for UI components
- RxJS 7.8.2 for reactive programming
- ngx-translate 15.0.0 for internationalization
- ng2-file-upload, ngx-clipboard for utilities

## 4. Architecture

The application follows a **traditional monolithic client-server architecture** with clear separation of concerns:

- **Backend Layer**: Express.js RESTful API server with middleware pipeline, using MVC-like pattern with Sequelize models
- **Frontend Layer**: Angular SPA (Single Page Application) with component-based architecture and services for API communication
- **Data Layer**: SQLite database accessed via Sequelize ORM
- **Security Layer**: Custom security middleware with challenge-based vulnerability detection

The architecture is **layered**, but routing is **mixed**: **Request → Security Middleware → (Explicit Route Handlers \*or\* Auto-generated Model-backed REST) → Business Logic/Model Hooks → Database**.

## 5. Key Components

**Backend Modules:**

- `app.ts` - Application bootstrap and dependency validation
- `server.ts` - Express server configuration, middleware setup, route registration
- `models/` - Sequelize ORM models (User, Product, Basket, Challenge, Feedback, etc.)
- `routes/` - API route handlers for authentication, orders, payments, file uploads, etc.
- `lib/` - Utility functions, security helpers, anti-cheat mechanisms, logging
- `data/` - Database seeding and static data (products, locales)

**Frontend Modules:**

- `frontend/src/main.ts` - Angular application bootstrap
- `frontend/src/app/` - Components, services, guards, routing
- Services for API communication (BasketService, ProductService, UserService, etc.)
- Guards (AdminGuard, LoginGuard, AccountingGuard, DeluxeGuard) for route protection

**Configuration:**

- `config/default.yml` - Application settings, product catalog, challenge configuration
- `tsconfig.json` - TypeScript compilation configuration

## 6. Data Flow

1. **User Interaction**: User interacts with Angular frontend UI
2. **HTTP Request**: Angular service sends HTTP request via RequestInterceptor
3. **Express Middleware Pipeline**: Request passes through security middleware (CORS, helmet, rate limiting, authentication/authorization checks)
4. **Route Handler**: Specific route handler processes the request
5. **Challenge Verification**: Security challenges evaluated based on request patterns
6. **Database Operations**: Sequelize models query/modify SQLite database
7. **Response**: Response flows back through middleware and returned to frontend
8. **UI Update**: Angular component updates view with response data

**WebSocket Flow**: Real-time events (notifications, challenge updates) flow via Socket.io between server and connected clients

## 7. Entry Points

**Development:**

- `npm run serve:dev` - Runs both backend (ts-node-dev) and frontend (ng serve) concurrently

**Production:**

- `npm start` - Runs compiled backend from `build/app.js`
- `node build/app` - Alternative production start command

**Build:**

- `npm run build:server` - Compiles TypeScript to JavaScript via tsc
- `npm run build:frontend` - Builds Angular frontend
- `npm run postinstall` - Installs dependencies and builds both frontend and backend

**Key Entry Files:**

- `app.ts` - Backend application entry point
- `frontend/src/main.ts` - Frontend Angular bootstrap
- `server.ts` - Server initialization and export
